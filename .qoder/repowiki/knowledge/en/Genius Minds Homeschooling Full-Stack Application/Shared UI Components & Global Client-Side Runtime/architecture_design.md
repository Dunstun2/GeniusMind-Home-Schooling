- `components/navbar.html` and `components/footer.html` are pure HTML fragments served at `/components/navbar.html` and `/components/footer.html`; each page places `<div id="navbar-placeholder">` / `<div id="footer-placeholder">` markers that `loadComponents()` replaces via `fetch(...).text()` + `outerHTML`, giving a server-side-include-like layout without a build step.
- `script.js` is the single client entry: it runs on `DOMContentLoaded`, calls `loadComponents()`, then `initializeGlobalUI()` (theme toggle persisted in `localStorage` under `data-theme`, mobile hamburger menu, scroll-shrink navbar, active-link highlighting, year injection), then dispatches to page-specific loaders (`loadAboutPageData`, `loadCoursesPageData`, `loadFaqPageData`, `loadBlogPageData`, `loadBlogPostDetailData`) which call REST endpoints under `/api/*` (`/api/content`, `/api/team`, `/api/courses`, `/api/faqs`, `/api/blog`, `/api/blog/:id`, `/api/settings`, `/api/social-media`, `/api/banners`).
- A dynamic hero/highlights slider (`initHighlightsSlider`) consumes `/api/banners` and reuses the same slide/dot/CTA pattern; CTA text is resolved through an optional `window.BannerCTA` bridge.
- `styles.css` owns all shared CSS — CSS custom properties under `:root` with a `[data-theme="dark"]` override block, glass-card utility, button variants, navbar/scrolled states, footer grid, social brand hover colors, and responsive breakpoints. No framework or preprocessor is used.
